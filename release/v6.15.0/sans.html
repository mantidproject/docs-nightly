Table of Contents

New Features¶

  • (#40443) Add option in TOML UserFiles and ISIS SANS GUI to compute wide-angle corrections on transmission data.

  • (#40253) Add the option to calculate the efficiency for an analyzer flipper in FlipperEfficiency v1 and in SANSISISPolarizationCorrections v1.

  • (#40585) New option for selecting a range of phi values in the masking tab of ISIS SANS GUI to generate multiple phi slices out of a single 1D reduction.

  • (#40585) Add option in TOML UserFiles to select a range of phi values to generate multiple phi slices in a single 1D reduction.

  • (#40253) Disable line edits for not needed runs on either calibration or correction for the algorithm dialog of SANSISISPolarizationCorrections v1.

  • (#40411) Setting a wrong value on the line edits of the settings tab in the SANS User Interface no longer creates a duplicate warning box when pressing enter.

  • (#40429) Added GenerateFlatCellWorkspaceLOQ for creating a flatcell workspace for ISIS SANS reduction of LOQ. The algorithm also generates an xml file with the detector ids to mask.

Bugfixes¶

  • (#40353) The ISIS SANS Interface now sends an appropriate error message when a directory is set as an user file path, the interface opens normally after the error is displayed.

  • (#40242) In the ISIS SANS workflow, the default number of cylindrical slices used in the calculation of the solid angle correction performed by the Q1D and Qxy algorithms has been changed from 10 to 11. This resolves a bug where unexpected results were seen for certain detector orientations and geometries. The number of cylindrical slices used in the ISIS SANS workflow can be changed from the user file.
